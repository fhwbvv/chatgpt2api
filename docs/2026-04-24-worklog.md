### 2. Updated packaging automation policy

Changed the automation behavior so packaging does **not** happen for every upstream `main` update.

New behavior:

- upstream sync job only follows formal upstream release tags
- if upstream has no release, sync is skipped
- if the release tag already exists in the fork, sync is skipped
- FreeBSD packaging only runs on:
  - tag push
  - manual workflow dispatch

Related files:

- `.github/workflows/sync-upstream.yml`
- `.github/workflows/build-freebsd.yml`

### 3. Restored serv00 and FreeBSD source deployment compatibility

After syncing to the new upstream backend, the source deployment path on serv00 broke because the newer code reintroduced direct `curl_cffi` usage in several modules.

Compatibility was restored by:

- routing HTTP client imports back through `services/http_session.py`
- preserving fallback to `requests` when `curl_cffi` is unavailable

Related commit:

- `f018cf5` `fix: restore freebsd source deployment compatibility`

### 4. Removed hard native build requirement for Pillow and tiktoken on FreeBSD

The serv00 deployment failed while building `Pillow`, and `tiktoken` was also a likely future native-build failure point.

To keep source deployment practical on serv00:

- removed `Pillow` from `requirements-freebsd.txt`
- removed `tiktoken` from `requirements-freebsd.txt`
- added pure-Python image MIME and size fallback logic
- added tokenizer fallback logic when `tiktoken` is not installed

Related commit:

- `3d62c02` `fix: avoid native pillow and tiktoken on freebsd`
