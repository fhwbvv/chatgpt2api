## Completed Work

### 1. Image API delivery improvements

Adjusted image generation and image edit responses so the frontend can use URL-based image delivery instead of relying on large inline Base64 payloads.

Key changes:

- Added generated image persistence under `data/generated-images`
- Added `response_format=url` support for:
  - `POST /v1/images/generations`
  - `POST /v1/images/edits`
- Added file serving route:
  - `GET /v1/images/files/{image_name}`

Related files:

- `services/api.py`
- `services/image_service.py`
- `services/chatgpt_service.py`
- `services/generated_image_store.py`

### 2. Image frontend usability updates

Updated the image page so the web UI no longer depends on long `b64_json` output for the main display flow.

Key changes:

- Frontend requests now prefer `response_format: "url"`
- Image history supports both:
  - URL-based results
  - legacy `b64_json` records
- Added direct preview and download support in the image results area

Related files:

- `web/src/app/image/page.tsx`
- `web/src/app/image/components/image-results.tsx`
- `web/src/lib/api.ts`
- `web/src/store/image-conversations.ts`

### 3. Source-tree static frontend hosting

Adjusted backend static asset resolution so source deployments can serve the frontend directly from:

- `web/out`

This keeps packaged releases working through `web_dist`, while allowing source deployments such as serv00 to work without extra manual copying.

Related file:

- `services/api.py`

### 4. FreeBSD and serv00 frontend build fix

Next.js 16 default build behavior caused FreeBSD build failures because Turbopack is not supported on `freebsd/x64`.

Fixed by switching the frontend production build script to Webpack:

- `next build --webpack`

Related file:

- `web/package.json`
